üö® Line Ending Issues

Windows uses CRLF (\r\n) while Linux uses LF (\n) for line endings.

To avoid issues:

‚Ä¢ Configure Git: git config --global core.autocrlf input

‚Ä¢ Use WSL for all development, avoid editing in Windows

‚Ä¢ If you must use Windows editors, ensure they're set to use LF endings

üö® File Permissions

Windows doesn't handle Linux file permissions well.

Best practices:

‚Ä¢ Keep your code in the WSL filesystem: /home/username/projects

‚Ä¢ Avoid storing code in Windows paths like /mnt/c/

‚Ä¢ If you see permission errors, use: chmod +x filename

üí° Accessing Windows Files from WSL

‚Ä¢ Windows C: drive is at: /mnt/c/

‚Ä¢ Your Windows user folder: /mnt/c/Users/YourWindowsUsername/

‚Ä¢ But remember: keep development files in WSL for best performance!
